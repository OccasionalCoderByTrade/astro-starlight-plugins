# Publishing Workflow

This document explains how the `cannoli-starlight-plugins` package is built and published to npm.

## Project Structure

```
project/
├── src/
│   ├── index.ts                              # Main entry point (re-exports both plugins)
│   ├── plugins/
│   │   ├── starlight-index-only-sidebar.ts  # Starlight plugin
│   │   └── rehype-validate-links.ts          # Rehype plugin
│   └── content/docs/                         # Demo site content (not published)
├── dist/                                     # Compiled library output (generated by tsup)
│   ├── index.js & index.d.ts
│   └── plugins/
├── site-dist/                                # Astro site build output (not published)
├── tsup.config.ts                            # Build configuration
├── package.json                              # Package metadata with exports
└── README.md                                 # User-facing documentation
```

## Build Tools

### tsup
- **Purpose**: Zero-config bundler for TypeScript packages
- **Config**: `tsup.config.ts`
- **TypeScript Config**: `tsconfig.build.json` (optimized for fast builds with `skipLibCheck: true`)
- **Output**: ESM modules with TypeScript type declarations
- **Why tsup**: Minimal configuration, perfect for Astro/Vite ecosystem packages, automatic `.d.ts` generation

### Package Configuration
- **Exports**: Subpath exports configured in `package.json` for flexible imports
  - `"."` → main entry (all plugins)
  - `"./starlight-index-only-sidebar"` → individual plugin
  - `"./rehype-validate-links"` → individual plugin
- **Files**: Only `dist/` is published (excludes source and demo content)

## Build & Publish Workflow

### Step 1: Build the Library
```bash
npm run build:lib
```

This runs `tsup` which:
1. Compiles `src/index.ts` to `dist/index.js` (+ `.d.ts`)
2. Compiles each plugin in `src/plugins/` to `dist/plugins/` (+ `.d.ts`)
3. Bundles shared dependencies into chunk files
4. Outputs ESM-only modules (no CommonJS)

**Output files:**
- `dist/index.js` & `dist/index.d.ts`
- `dist/plugins/starlight-index-only-sidebar.js` & `.d.ts`
- `dist/plugins/rehype-validate-links.js` & `.d.ts`
- `dist/chunk-*.js` (bundled dependencies)

### Step 2: Build the Demo Site (Optional)
```bash
npm run build
```

This:
1. Runs `npm run build:lib` automatically (via npm scripts)
2. Syncs docs to public directory
3. Builds the Astro demo site to `site-dist/`

The demo site is **not published** to npm (excluded by `files` field in `package.json`).

### Step 3: Verify Package Contents
```bash
npm pack --dry-run
```

Shows exactly what would be published:
- Compiled code in `dist/`
- Plugin source files in `src/`
- README, package.json

**Should NOT include:**
- Demo site content (`site-dist/`)
- Node modules
- `.astro/` files
- Build artifacts

### Step 4: Publish to npm
```bash
npm publish
```

This:
1. Automatically runs `prepublishOnly` script (rebuilds library)
2. Validates package contents against `files` field
3. Uploads to npm registry

### Step 5: Verify on npm
```bash
npm info cannoli-starlight-plugins
```

Check that the published version is available and has correct metadata.

## Important Notes

### TypeScript Declarations
- tsup automatically generates `.d.ts` files with `dts: true`
- Types are distributed alongside compiled JavaScript
- Users get full IDE autocomplete and type checking

### ESM Only
- tsup outputs only ESM (no CommonJS)
- Starlight plugins require ESM anyway
- Users must use `import` statements (no `require()`)

### Dependency Management
- **Runtime dependencies** in `dependencies`:
  - `yaml` — used by sidebar plugin
  - `glob` — used by link validation plugin
  - `unist-util-visit` — used by rehype plugin
- **Peer dependencies** in `peerDependencies`:
  - `@astrojs/starlight` — required for sidebar plugin
  - `astro` — required for integration
- **Dev/Type dependencies** in `devDependencies`:
  - `tsup` — build tool
  - `hast`, `vfile` — type definitions only

### Version Management
When updating the package before publishing:
1. Update `version` in `package.json`
2. Update `CHANGELOG.md` (recommended)
3. Commit changes
4. Create git tag matching version: `git tag v0.1.0`
5. Run `npm publish`

### Separation of Concerns
- **`dist/`**: npm package output (published)
- **`site-dist/`**: Astro demo site (not published)
- **`.gitignore`**: Both directories are ignored, allowing clean rebuilds

## Troubleshooting

### Build fails with TypeScript errors
```bash
npm run build:lib
```
Check error output for type issues. tsup uses the same `tsconfig.json`, so run `npm run check` to validate types first.

### Package includes unwanted files
Update the `files` field in `package.json` to be more specific. Test with:
```bash
npm pack --dry-run
```

### Build is slow
The build uses `tsconfig.build.json` with optimizations to keep it fast:
- `skipLibCheck: true` — skips type-checking dependencies (saves time checking node_modules)
- `include: ["src"]` — only type-checks your source files, not the entire project
- Ignores strict checking on heavy devDependencies like Astro, sharp, rehype packages

Without these optimizations, TypeScript would type-check all ~2000 dependency type definitions, causing 5+ minute builds. This config keeps it under 1 minute.
